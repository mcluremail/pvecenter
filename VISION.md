# PVECenter Vision & Roadmap

## Основная идея

PVECenter **не является заменой интерфейсу Proxmox VE**.

PVECenter --- это **единая точка управления территориально
распределёнными инфраструктурами Proxmox VE**, которая предоставляет
возможности, отсутствующие в отдельных веб-интерфейсах PVE.

### Основные принципы

-   Не вмешиваться в работу PVE.
-   Использовать официальный API Proxmox.
-   Агрегировать информацию.
-   Предоставлять единый обзор инфраструктуры.
-   Оставаться быстрым desktop-приложением.

------------------------------------------------------------------------

# Принципы

## Не копировать PVE

Не пытаться повторить весь GUI Proxmox. Реализовывать функции,
востребованные при работе с десятками серверов.

## Один источник правды

Каждый PVE остаётся владельцем своих данных.

PVECenter: - получает данные; - агрегирует; - кэширует; - отображает; -
выполняет операции через API.

## Минимум магии

Перед потенциально опасными действиями пользователь должен понимать: -
что будет сделано; - какие запросы будут отправлены; - какие последствия
ожидаются.

------------------------------------------------------------------------

# Desktop Mode

Самодостаточный режим работы.

Особенности: - без серверной части; - прямое подключение к PVE; -
API-токены хранятся локально; - кэш состояния; - периодический soft
update; - простое развёртывание.

------------------------------------------------------------------------

# Server Mode (опционально)

Сервер дополняет Desktop, а не заменяет его.

Функции: - агрегирование информации; - централизованный кэш; -
история; - события; - аудит; - уведомления; - REST API / WebSocket.

Пример компонентов:

-   Inventory
-   Cache
-   Scheduler
-   History
-   Event Bus
-   REST API
-   WebSocket

------------------------------------------------------------------------

# Архитектура клиента

Desktop → Core → Data Provider → Proxmox Adapter

В дальнейшем Data Provider сможет работать как напрямую с PVE, так и
через сервер.

------------------------------------------------------------------------

# Внутренняя модель данных

Не использовать JSON Proxmox внутри приложения.

Основные сущности: - Datacenter - Site - Cluster - Node - VM -
Container - Storage - Network - Task - Snapshot - Backup - User - Tag

------------------------------------------------------------------------

# Кэш

Функции: - уменьшение количества запросов; - быстрый UI; - soft
update; - хранение предыдущего состояния; - определение изменений.

------------------------------------------------------------------------

# Dashboard

Показывать: - узлы; - VM/LXC; - CPU; - RAM; - Storage; - Offline
Nodes; - Running Tasks; - Alerts.

------------------------------------------------------------------------

# Глобальный поиск

Поиск по: - VMID; - имени; - тегам; - владельцу; - IP; - Node; -
Cluster; - Storage.

------------------------------------------------------------------------

# Массовые операции

-   Start / Stop / Shutdown
-   Snapshot
-   Backup
-   Migrate
-   Clone
-   Delete
-   Move

------------------------------------------------------------------------

# История и события

Хранить: - запуск/остановку; - миграции; - ошибки; - резервные копии; -
изменения конфигурации.

События: - Node Online - VM Started - Task Finished - Migration
Completed - Backup Failed

------------------------------------------------------------------------

# Уведомления

Поддержка: - Desktop; - Telegram; - Email; - Webhook; - Matrix; -
Mattermost.

------------------------------------------------------------------------

# Policy Engine

Отдельный модуль, принимающий решения на основе состояния
инфраструктуры.

Этапы: 1. Рекомендации. 2. Полуавтоматическое выполнение (Approve →
Execute). 3. Полностью автоматические политики.

Возможные политики: - Affinity / Anti-Affinity; - ограничения на
backup; - миграции по расписанию; - разделение Production/Test; -
рекомендации по балансировке нагрузки (аналог DRS).

------------------------------------------------------------------------

# Аналитика

Хранить историю и строить: - тренды CPU; - тренды RAM; - рост Storage; -
uptime; - частоту миграций; - частоту ошибок.

------------------------------------------------------------------------

# Plugin Architecture

Core: - Inventory - Objects - Cache - Events - API

Плагины: - Proxmox - PBS - Ceph - Notifications - Reports - Policies -
DRS - Prometheus - Redfish

------------------------------------------------------------------------

# Безопасность

Не хранить токены в конфигурационных файлах.

Использовать: - KWallet; - GNOME Keyring; - Windows Credential Manager.

------------------------------------------------------------------------

# Долгосрочная цель

PVECenter --- это не просто GUI для Proxmox.

Это центр управления распределённой инфраструктурой, который: -
агрегирует данные; - упрощает работу администратора; - предоставляет
единый обзор; - постепенно добавляет автоматизацию без потери контроля.

------------------------------------------------------------------------

# Три ключевые идеи

1.  Опциональная серверная часть.
2.  Собственная внутренняя модель данных.
3.  Policy Engine как фундамент будущей автоматизации.

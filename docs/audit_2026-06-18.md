## Исправлено в этом аудите

### B1 — `executor.map` без `list()` (критично)
**Файл:** `ui/api/metrics.py:362, 457`
**Симптом:** вкладки Snapshots и VM Disks на storage были пустыми или неполными.
**Причина:** `executor.map()` возвращает ленивый итератор. Без `list()` futures не создаются, потоки не стартуют. Воркеры `fetch_vm_snapshots` и `fetch_vm_config` фактически не выполнялись.
**Fix:** `list(executor.map(...))` в обоих местах.

### B3+B4 — `ClusterTasksWorker` не в `_workers` + мёртвая ветка `gen is None`
**Файл:** `ui/mainwindow.py:740-756`
**Причина:** `ClusterTasksWorker` запускался через `threading.Thread` в обход `_run_worker`, не учитывался в `MAX_WORKERS`, не подключался `finished` signal. При падении до emit воркер утекал. Ветка `gen is None` в `_on_cluster_tasks_loaded` была мёртвой (никогда не достигалась).
**Fix:** регистрация в `_workers` + подключение `finished` → `_discard_worker`. Убрана мёртвая ветка.

### B5 — freeze-детектор пишет в stdout
**Файл:** `ui/mainwindow.py:916-933`
**Причина:** `print(...)` + `traceback.print_stack()` вместо `logger.error(...)`. При запуске не из терминала вывод терялся.
**Fix:** `logger.error(...)` + `traceback.print_stack(frame, file=buf)` → `logger.error("...\n%s", buf.getvalue())`.

### B6 — мёртвый `_watchdog_timer` (no-op)
**Файл:** `ui/mainwindow.py:160-164`
**Причина:** `QTimer` с `timeout.connect(lambda: None)` — no-op, не обновлял `_last_heartbeat` (это делал `_heartbeat_timer`). Мёртвый код.
**Fix:** удалён полностью.

### M3 — dead code comments в config.py
**Файл:** `config.py:358-361`
**Fix:** удалены комментарии `# Dead code below (kept for reference):` с упоминанием несуществующих функций.

### S1 — hardcoded русский без `tr()` + баг M8
**Файл:** `ui/mainwindow.py` (22 строки), `ui/i18n.py` (85 новых записей)
**Причина:** 22 user-facing строки на русском без `tr()` — при переключении языка не переводились. Бонусом найден баг M8: `error="Ошибка" in msg` — проверка по подстроке русского слова в уже переведённом сообщении, ломается при смене языка. И баг в `_on_host_remove`: сравнение `item_name == "Кластеры"` (хардкод) вместо `tr("Clusters")` — при локализации удаление кластеров/хостов молча отваливалось.
**Fix:** все строки обёрнуты в `tr()` с плейсхолдерами `{}`. `error=tr("Error") in msg`. Сравнения заменены на `tr("Clusters")` / `tr("Standalone hosts")`. Добавлены переводы для 17 новых ключей во все 5 языков (`_RU`, `_AR`, `_ZH`, `_FR`, `_ES`).

### O2 — linear search O(N) заменён на dict lookup O(1)
**Файлы:** `ui/utils.py` (+25 строк), `ui/mainwindow.py` (8 правок), `ui/tree_panel.py` (6 правок), `ui/detail_panel.py` (27 правок)
**Причина:** 31 место `next((c for c in self.nodes_cfg if c["name"] == host_name), None)` и 7 мест `next((v for v in self.all_vms if ...), None)` — linear search O(N). На 500 ВМ × ~50 вызовов × каждый refresh = ~25k сравнений.
**Fix:** добавлены helper'ы `build_cfg_index()`, `build_vm_index()`, `find_node_cfg()`, `find_vm()` в `utils.py`. Индекс `_cfg_by_name` строится в `__init__` и инвалидируется при изменении `nodes_cfg` (3 места). Индекс `_vms_by_key` строится инкрементально в `on_worker_finished` и перестраивается в `on_soft_refresh_result` / `update_data` / `set_lists`. ~50 dict lookup O(1) вместо ~25k сравнений.

### M4 — `os.chdir(parent_dir)` в `__main__.py`
**Файл:** `pve_center/__main__.py`
**Причина:** менял рабочую директорию процесса на родительскую без необходимости — все пути в коде используют абсолютные.
**Fix:** `os.chdir` удалён, файл сокращён с 12 до 7 строк.

### Бонус: ширина колонок `datacenter_summary` "плыла"
**Файл:** `ui/detail_panel.py:1378-1395, 1467-1481`
**Причина:** одна таблица `datacenter_summary` переиспользовалась между `_show_cluster_folder` (5 колонок) и `_populate_host_summary` (6 колонок). `setColumnCount(N)` не сбрасывает `setSectionResizeMode` для колонок, оставшихся от предыдущей конфигурации.
**Fix:** `setSectionResizeMode(QHeaderView.Interactive)` без индекса (сбрасывает все) перед установкой нужных режимов в обоих методах.

## Race conditions

### R1 — `self.all_vms` может измениться во время итерации
**Файл:** `ui/mainwindow.py`, `ui/detail_panel.py`
**Описание:** signal-хендлеры в Qt main thread, но `QMessageBox.exec()` вкладывает event loop. В этот момент может прилететь `QTimer.singleShot(1500, self.refresh_data)` и модифицировать `all_vms` во время итерации в другом хендлере.
**Вероятность:** низкая (требует modal dialog + timer одновременно).
**Fix:** копировать список перед итерацией или использовать generation-guard.
**Приоритет:** низкий.

### R2 — `_soft_refresh_running` + `_soft_gen` неатомарны
**Файл:** `ui/mainwindow.py:596-635`
**Описание:** `soft_refresh` проверяет `if self._soft_refresh_running:` и потом устанавливает `= True`. Между ними через nested event loop может выстрелить второй `soft_refresh`. Двойной набор воркеров. `_soft_gen` инкремент спасает от двойного обновления данных, но воркеры запускаются зря.
**Fix:** флаг-генерация или `QTimer` с `setSingleShot(True)`.
**Приоритет:** низкий.

### R3 — `_loading_hosts` set в tree_panel без синхронизации
**Файл:** `ui/tree_panel.py`
**Описание:** `_build_tree` и `update_node_statuses` оба модифицируют `self._loading_hosts`. Race только при nested event loops.
**Приоритет:** низкий.

### ARCH-4 — Хрупкая эвристика определения типа storage
**Файл:** `ui/tree_panel.py:753`
```python
if val and (val.count(".") > 0 or "/" in val):
    data["host_name"] = val   # standalone хост
else:
    data["cluster"] = val     # кластер
```
Кластер с именем `"my.cluster"` определится как хост. Standalone-хост `"proxmox"` (без точек) определится как кластер.
**Fix:** хранить тип явно в `ITEM_KEY_ROLE` кортеже, а не угадывать по имени.
**Приоритет:** средний.

## 🔴 Ошибки (баги, могут крашить или давать неверный результат)

### E1. `__init__.py` — версия не обновляется
`pve_center/__init__.py:1` — `__version__ = "0.1.2"`, хотя в `pyproject.toml` уже 1.4.0. Если кто-то импортирует `pve_center.__version__` — получит устаревшее значение.

### E2. `_on_cluster_tasks_loaded` — дублированная проверка
`mainwindow.py:887-891`:
```python

    if gen != self._tasks_gen:
        return
    if gen != self._tasks_gen:   # ← дубликат, бессмысленный

Вторая проверка — мёртвый код, copy-paste ошибка.

### E3. `on_worker_finished` — `worker` может быть None, но обращаемся к `worker.node_cfg`
`mainwindow.py:624`:
```python
is_cluster = worker.node_cfg.get("cluster_rep", False) if worker else False
```
Если `worker` is None — `False`. Но `worker` никогда не None здесь (передаётся из лямбды). Аналогично line 657. Безвредно, но `if worker` даёт ложное впечатление что None возможен.

### E4. `on_worker_finished` — `data["status"]` без проверки ключа
`mainwindow.py:623`:
```python
if data["status"] == "ok":
```
Если `FetchWorker` упадёт с исключением до emit, он эмитит `{"status": "error", "error": ...}`. Но если лямбда `on_worker_finished` получит `data` без ключа `status` (гипотетически) — KeyError. На практике не случится, но defensiveness не помешает.

### E5. `_on_import_config` — `self.nodes_config` вместо `self.nodes_cfg`  ⚠️ РЕАЛЬНЫЙ БАГ
`mainwindow.py:271`:
```python
self.nodes_config = merged
```
Присваивается `self.nodes_config`, а везде дальше используется `self.nodes_cfg`. После импорта конфиг не обновляется в `self.nodes_cfg` — следующий `refresh_data()` будет использовать старый список.

### E6. `import_config` при `merge=True` не устанавливает `_password`  ⚠️ ПОТЕНЦИАЛЬНАЯ ПОТЕРЯ ШИФРОВАНИЯ
`config.py:263-276`:
```python
if not merge:
    _password = password
    save_config(imported)
    return imported

current = load_config()
...
merged = list(current_map.values())
save_config(merged)   # ← _password НЕ обновлён!

```
При `merge=True` (единственный вызов из UI), `_password` НЕ устанавливается перед `save_config(merged)` — `save_config` использует старый кешированный пароль. Если пароль импортируемого файла совпадает — ОК. Если нет — `save_config` запишет merged в **plaintext** (json), потому что `_password` мог быть None или от другого пароля.

### E7. `VmConsoleWorker` — `vv_path` не очищается при ошибке запуска subprocess (не FileNotFoundError)
`backend.py:936-988`: при `subprocess.TimeoutExpired` (нормальный случай — detached) cleanup-поток запускается. Но если `Popen` упадёт с `PermissionError`/`OSError` (не `FileNotFoundError`) — исключение выйдет за `try`, `finally` эмитит `finished`, но `.vv` файл утечёт во `/tmp`.

### E8. `safe_pct` в `_vm_tabs.py` vs `_table_utils.py` — дублирование
`_vm_tabs.py:453` определяет локальную `safe_int` + использует `safe_pct` из `_table_utils`. Логика идентична, но `safe_int` определена дважды (lines 453 и 535) — копипаст.

### E9. `update_action_buttons` — `resume` кнопка есть, `suspend` — нет
`vm_actions.py:3-10` — кнопки: start, shutdown, reboot, reset, stop, **resume**. Но `suspend` отсутствует в кнопках, хотя в контекстном меню дерева (`tree_panel.py:150`) и в confirm-диалогах (`_vm_tabs.py:125`) `suspend` обрабатывается. Несоответствие: action bar не имеет кнопки Suspend, но контекстное меню — имеет. Возможно намеренно, но стоит проверить.

### E10. `show_host_info` — `host_name` параметр это `node_name`, а не `host_name`
`_host_tabs.py:349-351`:
```python

Нейминг сбивает с толку, но логика работает. Стилистический баг нейминга.

### E11. `show_host_info` — `host_data` может быть `{}`  ⚠️ РЕАЛЬНЫЙ БАГ
`_host_tabs.py:434-436`:
```python

### E12. `show_storage_detail` — табы BACKUPS/DISKS/ISO/TEMPLATES не скрываются

## Общее описание

PVE Center — десктопная панель управления Proxmox VE на PySide6 (Qt6).
Светлая тема. Русскоязычный интерфейс. Аутентификация через API-токены (user-bound).

**Структура проекта:** `/home/taurus/pve_center/`

### Ключевые файлы

| Файл | Назначение |
|---|---|
| `main.py` | Точка входа |
| `run.py` | Запуск из любой директории |
| `backend.py` | Все воркеры (FetchWorker, DeleteVmWorker, CreateVmWorker, VmActionWorker, VmConsoleWorker, ClusterTasksWorker и др.) |

## Констрейнты

- **Никакой тёмной темы** — пользователь ненавидит
- **Шрифты**: Noto Sans (интерфейс), Terminus (моноширинные данные)
- **Русскоязычный интерфейс**
- **Кнопка Отмена — всегда справа**, кнопка действия — слева от неё
- **Три варианта CDROM**: «Не использовать носители» / «Физический привод» / ISO
- **API-токены** — единственный способ аутентификации
- **Форсированное удаление ВМ** (`force=1` по умолчанию)

## Текущее состояние (что сделано)

### Реализованные фичи

1. **Главное окно**: сплиттер (дерево | детали) + задачи кластера снизу, строка состояния, автообновление 20с
2. **Дерево навигации**: кластеры → хосты → пулы → ВМ, отдельные хосты, хранилища, секции с кнопкой «+»
3. **Панель деталей**: 18 вкладок (мониторинг, оборудование, параметры, история, ВМ хоста, пул, сводка, хранилища, бэкапы, диски, ISO, шаблоны, сеть, сервисы, снапшоты)
4. **Диалог создания ВМ**: 5 секций (Основное, CPU/память, Система, Диск, Сеть), ISO+HA+EFI, фильтрация storage по узлу
5. **Удаление ВМ**: правый клик → «Удалить ВМ» → диалог с чекбоксом подтверждения → `DELETE /nodes/{node}/qemu/{vmid}?force=1`
6. **Добавление сервера**: авто-создание API-токена через пароль, шифрование токенов
7. **Обновление токена**: правый клик по хосту → «Создать токен заново»
8. **Консоль SPICE**: запрос SPICE proxy через API, запись `.vv` файла, запуск `remote-viewer`
9. **Мягкое обновление**: фоновый рефреш каждые 20с без сброса выделения
10. **Тост-уведомления**: FadeToast с затуханием 4с, клик копирует текст
11. **Статус-бары в таблицах**: CPU/RAM прогресс-бары с QProgressBar
12. **Детектор зависания**: watchdog + freeze detector

### Текущие баги (исправлены в этой сессии)

1. **`_on_item_clicked`** — краш при клике на ВМ: распаковывал VM_KEY_ROLE как 2-tuple, а там 3-tuple. Исправлено на `host_name, vmid, _node = vm_key`
2. **`_nodes.sort()`** — мутировал `self.all_nodes` mainwindow при каждом открытии диалога. Заменён на `sorted()`
3. **`r.json()` double call** — в `backend.py:75` вызывался дважды, второй раз падал. Исправлено сохранением в переменную
4. **Мёртвая переменная** `vm_name` в контекстном меню ВМ — убрана
5. **Дублирующийся `import timedelta`** в tree_panel.py — убран
6. **Дублирующийся `import DeleteVmWorker`** — локальный импорт удалён (уже был на уровне модуля)
7. **Пустой `models.py`** — удалён

### Известные архитектурные проблемы

1. **`all_pools`** не инициализирован в `__init__` mainwindow, всегда доступ через `getattr(self, 'all_pools', [])`
2. **18 вкладок** в detail_panel — большинство скрыты (`setTabVisible(False)`), включаются по контексту. Код громоздкий.
3. **Эмодзи в кнопках** (▶ Старт, ⏻ Выкл) — на Linux рендерятся по-разному
4. **Нет подтверждения** для stop/reset — принудительные действия без диалога

## API Proxmox (используемые эндпоинты)

| Эндпоинт | Назначение |
|---|---|
| `GET /cluster/nextid` | Следующий свободный VMID |
| `GET /cluster/ha/groups` | Список HA групп |
| `POST /cluster/ha/resources` | Добавить ВМ в HA группу |
| `GET /cluster/resources` | Все ресурсы кластера |
| `GET /nodes/{node}/qemu` | Список QEMU ВМ |
| `GET /nodes/{node}/status` | Статус узла |
| `DELETE /nodes/{node}/qemu/{vmid}` | Удалить ВМ |
| `GET /nodes/{node}/storage` | Список storage узла |
| `GET /nodes/{node}/storage/{storage}/content?content=iso` | ISO в хранилище |
| `GET /nodes/{node}/tasks` | Задачи узла |
| `POST /nodes/{node}/qemu` | Создать ВМ |
| `POST /access/ticket` | Аутентификация по паролю (для создания токена) |
| `POST/PUT /access/users/{user}/token/{token_id}` | Создать API-токен |

## UI/UX замечания (из аудита)

### Хорошо
- Светлая тема, нормальный контраст
- Статус-точки (зелёный/красный/жёлтый) — стандарт PVE
- Ховер строк, тосты с затуханием
- Разделение на секции в диалоге создания ВМ
- Кнопка Отмена справа везде консистентно

### Что можно улучшить
1. **Сплиттер** — дерево 280px узковато на маленьких экранах
2. **Статус-бар** — хорошо бы добавить индикатор статуса обновления
3. **QSpinBox VMID** — бессмысленный диапазон 0-999999999, лучше QLineEdit с валидатором
4. **Эмодзи в кнопках** — на Linux хромает рендер, лучше иконками
5. **Подтверждение для stop/reset** — сейчас нет, а действия опасные
6. **Удаление хоста** — диалог с «Да»/«Нет» без чекбокса, в отличие от удаления ВМ

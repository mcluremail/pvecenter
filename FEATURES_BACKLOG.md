## F2. Хранение токенов в системном keyring
Задолбался вводить пароль. Токены (мастер-пароль для расшифровки nodes.enc) можно хранить в:
- **Linux**: GNOME Keyring / KWallet (через `secretstorage` или `keyring`)
- **Windows**: Windows Credential Manager (через `keyring`)
- **macOS**: Keychain (через `keyring`)

Библиотека `keyring` (Python) — кроссплатформенная, единый API для всех трёх.
Хранить мастер-пароль (или сами токены?) в keyring, чтобы не вводить при каждом запуске.

Варианты:
- **Вариант A**: хранить мастер-пароль в keyring, автоматически подставлять в `load_config()`
- **Вариант B**: хранить сами токены (host, user, token_name, token_value) в keyring, отказаться от nodes.enc
- **Вариант C**: гибрид — keyring для мастер-пароля, fallback на ручной ввод если keyring недоступен

Опционально: чекбокс "Remember password" в диалоге ввода пароля.

Не реализовывать пока — только идея.

# Аудит UX/UI — 2026-06-19

Полный аудит пользовательского интерфейса PVE Center после v1.1.4.
Проверено вручную по коду. Найдено 20 проблем, из них 4 high, 8 medium, 8 low.

---

## 🔴 HIGH — реальные баги

### H1. Hardcoded русские строки без `tr()`

**Файлы:**
- `pve_center/ui/mainwindow.py:887-888`
- `pve_center/ui/widgets/vm_task_history_widget.py:62`

**Проблема:**
```python
# mainwindow.py:887
self.status_label.setText(
    f"Хостов: {hosts_ok}/{hosts_total}  ВМ: {vms_count}  Обновлено: {now_str}"
)

# vm_task_history_widget.py:62
end_item.setText("running...")
```

Строка статус-бара на русском и "running..." в истории задач не обёрнуты в `tr()`.
При любом языке интерфейса кроме русского — неразборчивый текст.

**Фикс:**
```python
self.status_label.setText(
    tr("Hosts: {ok}/{total}  VMs: {vms}  Updated: {ts}").format(
        ok=hosts_ok, total=hosts_total, vms=vms_count, ts=now_str
    )
)
end_item.setText(tr("running..."))
```

Добавить ключи `"Hosts: {ok}/{total}  VMs: {vms}  Updated: {ts}"` и `"running..."` во все 5 JSON.

### H2. `traceback.print_exc()` без user-facing ошибки — 3 места

**Файлы:**
- `pve_center/ui/mainwindow.py:693-694` (soft_refresh)
- `pve_center/ui/detail_panel/__init__.py:209-213` (show_details)
- `pve_center/ui/detail_panel/_vm_tabs.py:518-521` (display_full_vm_info)

**Проблема:**
При исключении во время обновления/отображения traceback пишется в stderr,
пользователь видит либо ничего, либо generic "An error occurred while loading
information". Конкретная причина теряется. Администратор не может понять что
сломалось без запуска из терминала.

**Фикс:**
Показывать notification toast с кратким текстом ошибки (последние ~100 символов
exception message). Полный traceback оставить в stderr для разработчика.

```python
except Exception as exc:
    traceback.print_exc()
    self._notifications.show(
        tr("Error: {err}").format(err=str(exc)[:100]),
        error=True,
    )
```

### H3. Нет ручной кнопки "Обновить"

**Файл:** `pve_center/ui/mainwindow.py` (весь)

**Проблема:**
`refresh_data()` вызывается только автоматически (20-секундный таймер) или
после add/create/delete host/VM. Пользователь не может принудительно обновить
данные, если подозревает что отображается устаревшее состояние. Единственный
способ — закрыть и переоткрыть приложение.

**Фикс:**
Добавить кнопку "Обновить" в toolbar (или status bar) + `QShortcut("Ctrl+R")`
или `QShortcut("F5")`, подключённый к `self.refresh_data()`.

---

### H4. Нет keyboard shortcuts

**Файл:** `pve_center/ui/mainwindow.py` (весь)

**Проблема:**
Ни одного `QShortcut` / `QKeySequence` в приложении. Нет:
- `Ctrl+R` / `F5` — обновить
- `Ctrl+Q` — выход
- `Esc` — снять выделение / закрыть диалог
- `Ctrl+N` — добавить сервер
- `Del` — удалить выделенный элемент из tree

Для desktop-приложения это expected baseline. Мышка + меню — не достаточны
для опытных администраторов.

**Фикс:**
Добавить `QShortcut`-ы в `MainWindow.__init__`:

```python
from PySide6.QtGui import QShortcut, QKeySequence

QShortcut(QKeySequence("Ctrl+R"), self, activated=self.refresh_data)
QShortcut(QKeySequence("F5"), self, activated=self.refresh_data)
QShortcut(QKeySequence("Ctrl+Q"), self, activated=self.close)
```

## 🟡 MEDIUM — UX проблемы

### M1. Status colors: хардкод hex вместо `theme.Color`

**Файлы (~30 мест):**
- `pve_center/ui/detail_panel/_host_tabs.py` — строки 197-201, 358, 450-454, 467, 534, 563, 601, 615
- `pve_center/ui/detail_panel/_vm_tabs.py` — строки 508-512, 533
- `pve_center/ui/detail_panel/_constants.py` — `_progress_style()` строки 45-54
- `pve_center/ui/widgets/vm_task_history_widget.py` — строки 61, 68, 72
- `pve_center/ui/widgets/cluster_tasks_widget.py` — строки 207, 240-244
- `pve_center/ui/widgets/vm_pool_widget.py` — `_pstyle()` строки 80-85
- `pve_center/ui/widgets/vm_metrics_widget.py` — строки 59, 164, 166

**Проблема:**
`#22c55e` (success), `#ef4444` (danger), `#f59e0b` (warning) хардкодятся по всему
UI. В `theme.py` есть класс `Color` с константами `SUCCESS`, `DANGER`, `WARNING`,
но он используется только в `hover.py`. При добавлении dark mode все 30 мест
сломаются — придётся искать и заменять каждый хекс.

**Фикс:**
Вынести status colors в `_constants.py` (или использовать `Color.*` напрямую):

```python
# _constants.py
from ..theme import Color

STATUS_COLORS = {
    "success": Color.SUCCESS,
    "danger": Color.DANGER,
    "warning": Color.WARNING,
}
```

Заменить все `QColor("#22c55e")` → `QColor(STATUS_COLORS["success"])` и т.д.

### M2. Inline `setStyleSheet` с хардкод цветами — ~20 мест

**Файлы:**
- `mainwindow.py:103, 108-114, 280-281, 299`
- `tree_panel.py:87, 136-138, 229-233`
- `add_server_dialog.py:55, 68, 73, 78, 168-170`
- `create_vm_dialog.py:518`
- `detail_panel/__init__.py:77`
- `_host_tabs.py:358, 467`
- `_table_utils.py:97-99`
- `vm_config_editor_dialog.py:70, 74, 82`
- `vm_device_editors.py:103, 234, 315, 369, 607`
- `notification.py:31-38`

**Проблема:**
Inline-стили с hex-цветами игнорируют `theme.py` QSS. Цвета `#6b7280` (gray),
`#f1f5f9` (bg), `#334155` (text), `#c0392b` (red) хардкодятся. При смене темы
(например dark mode) эти элементы останутся светлыми.

**Фикс:**
Заменить на QSS class selectors (`.setProperty("class", "error-label")`) или
использовать `Color.*` константы. Для часто повторяющихся (hint-label, warning,
error) добавить QSS-правила в `theme.py`.

### M3. Empty tables без placeholder сообщения — 6+ мест

**Файлы:**
- `pve_center/ui/widgets/vm_hardware_widget.py:60` — hardware table
- `pve_center/ui/widgets/vm_options_widget.py:46-61` — options table
- `pve_center/ui/widgets/vm_pool_widget.py:38-65` — pool VMs table
- `pve_center/ui/detail_panel/_host_tabs.py:174-241` — cluster summary table
- `pve_center/ui/detail_panel/_host_tabs.py:435-469` — host VM table
- `pve_center/ui/detail_panel/_storage_tabs.py:226-258` — storage table

**Проблема:**
Когда данных нет, таблица остаётся пустой (0 строк) — выглядит как баг или
незагруженные данные. Пользователь не понимает: данных действительно нет, или
произошла ошибка, или ещё загружается?

**Фикс:**
Добавить placeholder-row когда `len(rows) == 0`:

```python
if not rows:
    table.setRowCount(1)
    placeholder = QTableWidgetItem(tr("No data"))
    placeholder.setFlags(Qt.NoItemFlags)
    placeholder.setTextAlignment(Qt.AlignCenter)
    table.setItem(0, 0, placeholder)
    return
```

Или использовать `setPlaceholderText` (Qt 6.x) если доступно.

### M4. Нет tooltips на action buttons

**Файл:** `pve_center/ui/detail_panel/__init__.py:66-84`

**Проблема:**
Кнопки Start / Shutdown / Reboot / Reset / Stop / Resume + Console — без tooltip.
Новичок не знает разницу между "Stop" и "Reset" (оба останавливают VM, но Reset
— аппаратный сброс без graceful shutdown). "Shutdown" vs "Stop" — тоже не
очевидно (Shutdown = ACPI signal, Stop = kill).

**Фикс:**
Добавить `setToolTip` на каждую кнопку:

```python
TOOLTIPS = {
    "start":   tr("Start VM"),
    "shutdown": tr("Send ACPI shutdown signal to VM"),
    "reboot":   tr("Send ACPI reboot signal to VM"),
    "reset":    tr("Hard reset VM (unsaved data will be lost)"),
    "stop":     tr("Force stop VM (kill process, unsaved data lost)"),
    "resume":   tr("Resume suspended VM"),
}
btn.setToolTip(TOOLTIPS[action_key])
self._console_btn.setToolTip(tr("Open SPICE/VNC console"))
```

### M5. VM actions: shutdown/reboot/suspend без подтверждения

**Файл:** `pve_center/ui/detail_panel/_vm_tabs.py:130-141`

**Проблема:**
Только `stop` и `reset` имеют confirmation dialog. `shutdown`, `reboot`,
`suspend`, `migrate` выполняются сразу по клику. Для desktop-приложения где
можно промахнуться мышкой (особенно на узкой action bar с 6 кнопками рядом) —
рискованно. `shutdown` может потерять unsaved state если гость не обрабатывает
ACPI правильно.

**Фикс:**
Расширить confirmation на все "опасные" actions. Минимум — `shutdown` (может
завершиться некорректно если ACPI не настроен). Опционально — `reboot`, `suspend`.

```python
CONFIRM_ACTIONS = {"stop", "reset", "shutdown", "reboot"}
if action in CONFIRM_ACTIONS:
    ...
```

Спорно: PVE web UI тоже не подтверждает shutdown/reboot. Но desktop мышка
менее точна чем web click. Решать пользователю.

### M6. Braille spinner без accessible text

**Файл:** `pve_center/ui/mainwindow.py:102-104, 124-128`

**Проблема:**
`_refresh_spinner` показывает unicode braille-символы `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` для
индикации обновления. Нет текста "Refreshing..." рядом. Screen readers
прочитают braille-точки как "braille pattern dots-1237" — бессмысленно.

**Фикс:**
Добавить `setAccessibleName` + текст-подсказку:

```python
self._refresh_spinner.setAccessibleName(tr("Refreshing data"))
# ИЛИ добавить текст рядом:
self._refresh_spinner.setToolTip(tr("Data refresh in progress"))
```

### M7. Нет контекстного меню на VM/host в tree

**Файл:** `pve_center/ui/tree_panel.py:102`

**Проблема:**
Контекстное меню (right-click) есть только для удаления. Нет быстрых actions:
"Start VM", "Shutdown VM", "Open console" через right-click на узле дерева.
Пользователю приходится кликать узел → ждать detail panel → искать action bar.

**Фикс:**
Добавить actions в контекстное меню для VM-узлов:

```python
if item_type == "vm":
    menu.addAction(tr("Start"), lambda: self.vm_action_requested.emit("start", ...))
    menu.addAction(tr("Shutdown"), lambda: self.vm_action_requested.emit("shutdown", ...))
    menu.addAction(tr("Console"), lambda: self.console_requested.emit(...))
    menu.addSeparator()
    menu.addAction(tr("Delete"), delete_action)
```

Нужен новый signal `vm_action_requested` в TreePanel.

### M8. `int(vmid_text)` без try/except в create VM

**Файл:** `pve_center/ui/create_vm_dialog.py:529`

**Проблема:**
```python
vmid = int(vmid_text) if vmid_text else 0
```

Хотя `vmid_line` имеет `QIntValidator(100, 999999999)`, валидатор не
гарантирует что текст валиден (пользователь может вставить из буфера).
`int()` упадёт с `ValueError` → uncaught exception → диалог не закроется
корректно.

**Статус:** Частично mitigated валидатором, но не полностью. Лучше оборачивать.

**Фикс:**
```python
try:
    vmid = int(vmid_text) if vmid_text else 0
except ValueError:
    vmid = 0  # PVE auto-assign
```

## 🟢 LOW — polish

### L1. `setFixedSize` на диалогах

**Файлы:**
- `add_server_dialog.py:16` — `self.setFixedSize(500, 420)`
- `mainwindow.py:272` — `dlg.setFixedSize(480, is_running and 280 or 240)`
- `mainwindow.py:356` — `dlg.setFixedSize(420, 130)`

На маленьких экранах (netbook 1366×768) `AddServerDialog` 500×420 может не
влезть с декорациями окна. Лучше `setMinimumSize` + `adjustSize()` для
авто-подгона под контент.

---

### L2. Button ordering: action перед Cancel

**Файлы:**
- `mainwindow.py:322-323` — `[Delete] [Cancel]`
- `add_server_dialog.py:101-110` — `[Add] [Cancel]`
- `create_vm_dialog.py:423-432` — `[Create] [Cancel]`
- `vm_config_editor_dialog.py:91-103` — `[Save] [Cancel]`
- `vm_device_editors.py:158-168, 243-251, 320-329, 411-421, 534-544, 611-621` — `[Save] [Cancel]`

На Windows convention — Cancel справа, action слева. На macOS/Linux — наоборот.
Текущий layout `[Action] [Cancel]` соответствует Windows. Спорно, но
посовательно по всему app — это style choice, не баг. Менять только если
хотим следовать GNOME convention (`[Cancel] [Action]`).

### L3. Hardcoded column widths

**Файлы:**
- `_host_tabs.py:184-188` — `setColumnWidth(1, 70)`, `setColumnWidth(3, 55)`
- `vm_pool_widget.py:20-28` — `setColumnWidth(1, 50)`, `setColumnWidth(2, 65)`
- `cluster_tasks_widget.py:148-149` — `setColumnWidth(0, 155)`, `setColumnWidth(1, 155)`

Column widths фиксированы в пикселях. На high-DPI (4K, 200% scale) могут быть
слишком узкие. Лучше `QHeaderView.Stretch` + `setSectionResizeMode` (частично
уже сделано в `col_specs`).

### L4. Manual text truncation вместо elide

**Файлы:**
- `cluster_tasks_widget.py:237` — `status[:30] + '…'`
- `_host_tabs.py:782` — `str(model)[:50]`

Ручная обрезка по символам. На разных шрифтах/размерах 30 символов = разная
ширина в пикселях. Лучше `QFontMetrics.elidedText(text, Qt.ElideRight, width)`.

---

### L5. Нет QAccessible

**Файл:** весь `pve_center/ui/`

Никаких `accessibleName` / `accessibleDescription`. Screen readers не смогут
озвучить action buttons, status indicators, spinner. Для desktop app с
потенциально enterprise-аудиторией (PVE admins, которые могут иметь
ограничения по зрению) — это gap. Но полная accessibility — большой effort.

---

### L6. Нет dark mode

**Файл:** `pve_center/ui/theme.py`

Только light theme. PVE web UI имеет dark mode. Многие admins работают ночью
и предпочитают dark. Но полная palette migration — большой effort, требует
рефакторинга всех 30 hardcoded color sites (M1) + 20 inline styleSheet sites (M2).

---

### L7. `vm_action_bar.setFixedHeight(32)`

**Файл:** `pve_center/ui/detail_panel/__init__.py:57`

На high-DPI 32px может быть мелко. Лучше `setMinimumHeight(32)` + expanding
size policy.

---

### L8. `storage_plot_widget.setFixedHeight(220)`

**Файл:** `pve_center/ui/detail_panel/_storage_tabs.py:93`

Plot фиксированной высоты. На больших экранах wasted space, на маленьких —
overflow. Лучше `setMinimumHeight(200)` + expanding policy.
